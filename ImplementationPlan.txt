 What Salesforce Gives You (Developer Console / Setup)                                                                                       
  
  - Run tests → wait → see results → gone after you navigate away                                                                               - No historical data — every run is fire-and-forget                                                                                         
  - No trend tracking — "is our code quality improving or declining?" — no answer
  - Coverage data is scattered across multiple Setup pages
  - No real-time progress — you submit and poll manually
  - Governor limits on Tooling API calls if you're querying programmatically
  - UI is clunky, slow, and not designed for test monitoring workflows

  The Key Architectural Win

  Browser → Spring Boot (cache + persist + WebSocket) → Salesforce API

  Spring Boot sits in the middle and:
  1. Caches test class lists and org stats so you're not burning API calls on every page load
  2. Persists every run to PostgreSQL — Salesforce doesn't keep this for you
  3. Polls asynchronously and pushes updates via WebSocket — no manual refreshing
  4. Aggregates coverage and pass rate data into trend charts that Salesforce simply doesn't offer
